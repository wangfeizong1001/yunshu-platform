## 三、核心模块设计

### 3.1 权限系统

采用 **RBAC**（Role-Based Access Control）模型：

```
User ──< Role ──< Permission
│          │          │
│          └── Menu  ─┘
│          │
│          └── Button Permission
│
└── Dept ──< Data Scope
```

**权限检查流程**：

```
1. 用户登录 → 获取 Token
2. 加载用户信息 + 角色列表
3. 加载菜单树（基于角色权限过滤）
4. 生成路由表（动态路由）
5. 按钮级权限（v-permission 指令）
```

### 3.2 安全中间件

#### 请求限流 (rateLimit)

```typescript
// packages/server-express/src/middlewares/rateLimit.ts
// - 基于 IP 的滑动窗口限流
// - 可配置每分钟最大请求数
// - 超出限制返回 429 Too Many Requests
```

#### 文件上传安全 (uploadGuard)

```typescript
// packages/server-express/src/middlewares/uploadGuard.ts
// - 白名单 MIME 类型校验
// - 文件大小限制
// - 文件扩展名校验
// - 文件名过滤（路径遍历防护）
// - 魔数校验（实际文件内容检查）
```

### 3.3 日志系统

**结构化日志格式**:

```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "level": "info",
  "module": "user.service",
  "message": "用户登录成功",
  "userId": "1001",
  "ip": "192.168.1.1",
  "traceId": "abc-123-def"
}
```

**支持的传输**（Transport）:
- Console（开发环境）
- File（文件日志，按日期轮转）
- Loki（集中式日志收集）

### 3.4 多租户架构

```
┌──────────────────────────────────────────────┐
│              平台层 (Platform)                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │  租户A   │  │  租户B   │  │  租户C   │   │
│  │ 套餐: 企业│  │ 套餐: 基础│  │ 套餐: 专业│  │
│  │ 用户数: 50│  │ 用户数: 10│  │ 用户数: 20│  │
│  └──────────┘  └──────────┘  └──────────┘   │
└──────────────────────────────────────────────┘

数据隔离策略：
1. 逻辑隔离（tenant_id 字段）
2. 权限隔离（每个租户仅访问自有数据）
3. 套餐限制（用户数/功能点/存储配额）
```


## 八、架构原则

1. **单一职责**：每个模块/组件只做一件事
2. **依赖倒置**：面向接口编程，而非实现
3. **开闭原则**：对扩展开放，对修改关闭
4. **最小化原则**：仅暴露必要的 API
5. **可观测性**：所有关键路径都有日志和监控
6. **安全优先**：认证、授权、输入校验无处不在
