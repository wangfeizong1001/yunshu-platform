## 前置准备

### 系统要求

- Node.js &gt;= 20.0.0
- pnpm &gt;= 9.0.0
- Docker &gt;= 20.10
- Docker Compose &gt;= 2.0

### 安装依赖

```bash
# 安装 pnpm
npm install -g pnpm@9

# 安装项目依赖
pnpm install
```

## 快速开始

### 使用 Docker Compose 一键启动

```bash
# 克隆项目
git clone <your-repo-url>
cd yunshu-platform

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

启动成功后，访问：
- 管理后台：http://localhost
- 后端 API：http://localhost:8080

## 环境配置

### 环境变量

复制 `.env.example` 为 `.env` 并修改配置：

```bash
cp .env.example .env
```

主要环境变量：

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| NODE_ENV | 环境类型 | development |
| DB_HOST | 数据库地址 | localhost |
| DB_PORT | 数据库端口 | 5432 |
| DB_NAME | 数据库名称 | yunshu |
| DB_USER | 数据库用户 | yunshu |
| DB_PASSWORD | 数据库密码 | yunshupassword |
| REDIS_HOST | Redis 地址 | localhost |
| REDIS_PORT | Redis 端口 | 6379 |
| VITE_API_BASE_URL | API 基础路径 | /api |

## Docker 部署

### 多阶段构建说明

项目使用多阶段构建优化镜像大小：

1. **Builder 阶段**：安装依赖、编译代码
2. **Runner 阶段**：仅包含运行时必需的文件

### 构建和运行单个镜像

#### 后端服务

```bash
# 构建镜像
docker build -t yunshu-backend .

# 运行容器
docker run -d \
  --name yunshu-backend \
  -p 8080:8080 \
  --network yunshu-network \
  --env-file .env \
  yunshu-backend
```

#### 管理后台

```bash
# 构建镜像
docker build -f apps/admin/Dockerfile -t yunshu-admin .

# 运行容器
docker run -d \
  --name yunshu-admin \
  -p 80:80 \
  --network yunshu-network \
  yunshu-admin
```

### 使用 Docker Compose 部署

#### 开发环境

```bash
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d
```

#### 生产环境

```bash
docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## CI/CD 流水线

### GitHub Actions 配置

项目已配置 GitHub Actions CI/CD 流水线，包含以下任务：

1. **Lint**：代码风格检查
2. **Test**：单元测试
3. **Type Check**：类型检查
4. **Build**：项目构建
5. **Docker Build & Push**：构建并推送 Docker 镜像（仅 main 和 develop 分支）

### 触发条件

- 推送到 `main` 或 `develop` 分支
- 向 `main` 或 `develop` 分支发起 Pull Request
- 手动触发（workflow_dispatch）

### 容器镜像

镜像推送到 GitHub Container Registry (GHCR)：

```
ghcr.io/<your-username>/yunshu-platform:<tag>
```

## 生产环境部署

### 安全建议

1. **修改默认密码**
   - MySQL root 密码
   - 应用数据库密码
   - Redis 密码（如需要）

2. **使用 HTTPS**
   - 配置 SSL 证书
   - 强制 HTTPS 跳转

3. **限制网络访问**
   - 使用防火墙规则
   - 仅暴露必要端口

4. **数据备份**
   - 定期备份 MySQL 数据
   - 备份 Redis 持久化文件

### 高可用部署

#### 使用 Docker Swarm

```bash
# 初始化 Swarm
docker swarm init

# 部署服务
docker stack deploy -c docker-compose.swarm.yml yunshu
```

#### 使用 Kubernetes

参考 `k8s/` 目录下的 Kubernetes 配置文件。

### 监控和日志

1. **日志收集**
   - 配置 ELK Stack
   - 或使用 Loki + Promtail + Grafana

2. **性能监控**
   - 使用 Prometheus + Grafana
   - 监控容器资源使用

## 常见问题

### 容器无法启动

```bash
# 查看容器日志
docker-compose logs <service-name>

# 检查端口占用
netstat -tulpn | grep <port>
```

### 数据库连接失败

```bash
# 检查 MySQL 容器状态
docker-compose ps mysql

# 进入 MySQL 容器
docker-compose exec mysql mysql -u yunshu -p
```

### 构建失败

```bash
# 清理构建缓存
docker builder prune

# 重新构建
docker-compose build --no-cache
```

### 性能优化

1. **启用 Gzip 压缩**（已在 nginx 配置中启用）
2. **配置 CDN** 加速静态资源
3. **使用 Redis 缓存** 热点数据
4. **数据库优化** 索引和查询
